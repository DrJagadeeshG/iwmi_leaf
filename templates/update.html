Important — Handle With Care

This sheet contains the raw data that powers the entire dashboard. Incorrect edits will directly affect feasibility scores, map visualizations, and all analysis outputs. Please review the risks below before making changes.

Mistake What happens How to avoid
Deleting or changing a BLOCK_ID Block disappears from the map or gets wrong data — no error shown Never edit columns A–E (BLOCK_ID, STATE_ID, DISTRICT_I, id, Block_name)
Deleting a row That block vanishes completely from the dashboard Only add rows, never delete existing ones
Entering text in a numeric column Variable shows as "No data" for that block; feasibility scores break All variable columns (F onwards) must be numbers only
Renaming a column header Variable stops working everywhere — map, filters, feasibility Never rename column headers; add new columns at the end instead
Leaving cells empty Block shows "No data" for that variable; may lower feasibility scores Use 0 instead of leaving blank if the value is zero
Duplicating a BLOCK_ID Duplicate rows cause unpredictable results for that block Each BLOCK_ID must appear exactly once
Adding a new variable column Column appears in data but won't show in dashboard until added to the Intervention Config sheet too After adding a new column here, add a matching row in the Variable Catalog (right side of Sheet 1)
Recommended: Before making bulk edits, download the current sheet as a backup (File > Download > CSV). If something breaks, you can restore from that backup.
What You Can Update

The Google Sheet has two sections side by side. The left side (columns A–H) controls interventions. The right side (columns I–N) defines the variable catalog.

Column What it controls How to fill
Cluster Intervention name — each unique value becomes a selectable intervention in the dashboard Type the intervention name. Repeat the same name on every row belonging to that intervention. e.g. Organic Farming, Solar Irrigation
I_variable Which data variable this row uses for feasibility scoring Use the variable code from the Variable Catalog (right side, column J). e.g. AD, C, BF
range_min Minimum acceptable value — below this, the block scores 0% for this variable A number. Check the dashboard for the data range of the variable. e.g. 20
range_max Maximum acceptable value — above this, the block scores 100% for this variable A number, must be greater than range_min. e.g. 60
Preference Whether higher or lower values are better for feasibility One of: higher, lower, or moderate higher = more is better, lower = less is better
I_weight Importance weight — how much this variable matters relative to others in the intervention A number, typically 1. Use higher values (2, 3) for more important variables. e.g. 1
I_label Display label shown in the dashboard filter panel Short human-readable name. e.g. % Paddy Area, Cattle density
I_description Tooltip description explaining why this variable matters for the intervention One sentence explanation. e.g. Moderate paddy area allows diversification.
To add a new intervention: Add new rows at the bottom. Put the intervention name in Cluster (e.g. "Solar Irrigation"), then fill one row per variable with I_variable, ranges, weight, label, and description. It will appear automatically in the dashboard dropdown.
To remove an intervention: Delete all rows with that Cluster name. The intervention disappears from the dashboard.
Do not leave the Cluster column empty in intervention rows — rows without a Cluster name are treated as variable-only metadata (right side).
Column What it controls How to fill
group Category grouping in the dashboard (e.g., the expandable sections in the filter panel) One of: Land & Agri, Water, Infrastructure, Livestock, People, Soil, Climate Must match exactly (case-sensitive)
subgroup Sub-category within the group Free text sub-heading. e.g. Cropping system, Irrigation, Livestock density
variable Variable code — the column name in the shapefile data. This is what I_variable references. Use the existing code. New codes must match a column in the block shapefile. e.g. A, B, C, AD, BF
Weight Default weight for this variable (used when no intervention-specific weight is set) A number, typically 1. e.g. 1
label Display name shown on the map and in variable selectors Short descriptive name. e.g. Farming Households (%), Cattle density (per 100 hectares)
Description Full description shown in tooltips and variable detail panels One-line explanation of the variable. e.g. Share of households with farming as major activity
Variable codes are fixed: The codes (A, B, C, AD, etc.) correspond to columns in the underlying shapefile. You can change labels and descriptions freely, but adding a completely new variable code requires updating the shapefile data too.
Do not change existing variable codes — this will break the link between the data and the dashboard. Only change labels, descriptions, groups, and subgroups.
How It Works
1

Edit the Google Sheet

Open the sheet and make your changes — add interventions, adjust ranges, update weights or labels. Changes are saved automatically in Google Sheets.

2

Wait ~1 minute

Google takes up to a minute to propagate changes to the published CSV. No action needed on your part.

3

Click "Refresh Now"

Hit the refresh button above to pull the latest data into the LEAF system. The dashboard will reflect your changes immediately.

4

Auto-sync every 5 minutes

Even without manual refresh, the system automatically picks up changes from the Google Sheet every 5 minutes.

Data Not Editable via Google Sheets

The following data is stored in geospatial files (shapefiles) and cannot be edited through Google Sheets. Contact the development team to update these.

Data What it contains Why it can't be in Sheets
Block boundaries Geographic boundaries of each block shown on the map Geospatial polygon data
GP boundaries & data Gram Panchayat level geometries and variable values Geospatial polygon data
District boundaries District outlines shown on the map Geospatial polygon data
Protected areas National parks, wildlife sanctuaries overlay Geospatial polygon data
AI knowledge base PDFs and documents used by the AI recommendation engine Stored in vector database (ChromaDB)
