Livestock Sub-filter

The Livestock dropdown's sub-categories (Dairy, Goatery, Piggery, Backyard_Poultry, Duckery, Fishery_Activity) have their own variable ranges and weights. These are now maintained directly in the Intervention & Variable Config (dss_input) Google Sheet — edit a range, weight or label there and it flows to the live app within ~5 minutes. No file upload, no redeploy.

StepWhat to do
1 Open the dss_input config Google Sheet (the same sheet that drives the interventions and variable ranges).
2 Find the sub-type rows: Cluster = the commodity name (Dairy / Goatery / …) and parent = Livestock.
3 Edit range_min / range_max / I_weight / I_label as needed. Keep all six commodities and keep parent = Livestock. Changes go live within ~5 min (or click Refresh).

Handle With Care

This sheet contains raw data that powers the entire dashboard. Wrong edits break feasibility scores, maps, and analysis.

Mistake Impact
Editing BLOCK_ID or ID columns Block disappears or gets wrong data
Deleting a row Block vanishes from dashboard
Text in numeric column Shows "No data", breaks scores
Renaming column header Variable stops working everywhere
Empty cells "No data" for that block/variable
Duplicate BLOCK_ID Unpredictable results
New column added Won't show until added to Intervention Config sheet too
Safe columns to edit: Column F onwards (variable values). Never touch columns A-E (BLOCK_ID, STATE_ID, DISTRICT_I, id, Block_name).
Backup first: File > Download > CSV before bulk edits.

AI Knowledge Base

Upload a policy or guideline PDF to the AI knowledge base. The document joins the retrieval pool and the knowledge base rebuilds automatically on the next AI Insights request.

Lazy rebuild: embedding the new document is deferred to the next AI recommendation, so the first AI request after an upload may take a little longer.
