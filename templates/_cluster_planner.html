Upload edited cluster CSV

The CSV must have one row per (cluster, village). Upload replaces all clusters in the current scope (block + commodity). Easiest path: download the current clusters, edit in Excel, save as CSV, upload here.

Expected columns

Column Required Notes
cluster_idYesReuse existing IDs, or any string for new clusters. Same ID = same cluster.
commodityYesOne of: Dairy, Goatery, Piggery, Backyard_Poultry, Duckery, Fishery_Activity
district_nameYesMatch case used in the village data (uppercase for MMUA seed).
block_nameYesMust match the scope you're uploading to.
vill_nameYesVillage display name.
gp_nameYesParent GP.
lat / longYesWGS84 decimal degrees. Centroid is recomputed on upload.
membersYesMembers in this village interested in the commodity. Total per cluster is recomputed.
pashu_sakhiNoAssigned Pashu Sakhi name.
block_coordinatorNoAssigned block coordinator name.

Common edits

  • Split a cluster: change one row's cluster_id to a new value.
  • Merge two clusters: give them the same cluster_id.
  • Drop a village: delete its row.
  • Assign people: fill in pashu_sakhi / block_coordinator on each row.
Tip: hit Download current as template below to get a CSV with valid IDs and coordinates already populated for your block / commodity. Then just edit and re-upload.
