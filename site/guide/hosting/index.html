    Hosting & Handover Guide¶

    This guide is for anyone who needs to host LEAF DSS independently — whether you're a government IT team, an NGO partner, a research institution, or a developer taking over the project. It covers what you need, your hosting options, estimated costs, and step-by-step instructions.

    What You're Hosting¶

    LEAF DSS is a single Python web application that serves both the interactive map UI and the REST API. There is no separate frontend deployment — everything runs from one process.

    System Requirements¶

    Resource Minimum Recommended
    CPU 1 vCPU 2 vCPUs
    RAM 512 MB 1–2 GB
    Disk 500 MB 1 GB
    Python 3.10+ 3.11 or 3.12
    OS Any Linux Ubuntu 22.04+

    RAM is the main constraint — the app loads shapefiles into memory (~10–15 MB cached). With the AI/RAG feature enabled, RAM usage increases to ~500 MB due to ChromaDB and embeddings.

    What's Included¶

    Component Description Required?
    Flask web app Map UI + REST API + Swagger docs Yes
    Shapefiles & CSV Block/GP/district/protected area geodata Yes
    MkDocs site Written documentation (pre-built HTML) Optional
    AI/RAG pipeline Policy document recommendations Optional
    PDF policy docs Source documents for AI Only if AI enabled

    Hosting Options & Costs¶

    Option 1: Render (Current Setup) — Recommended for Getting Started¶

    Render is a managed platform that auto-deploys from GitHub.

    Plan Monthly Cost RAM CPU Notes
    Free $0 512 MB Shared Sleeps after 15 min inactivity, cold starts ~30s
    Starter $7/month 512 MB Shared Always on, no sleep
    Standard $25/month 2 GB 1 vCPU Recommended for production
    Pro $85/month 4 GB 2 vCPUs For heavy AI/RAG usage

    Pros: Zero server management, auto-deploy from Git, free SSL, health checks. Cons: Free tier has cold starts; US/EU regions only.

    Setup Steps¶

    1. Push code to a GitHub repository
    2. Sign up at render.com
    3. Click New → Web Service → connect your GitHub repo
    4. Configure:
      • Build Command: pip install -r leaf_flask/requirements.txt && pip install mkdocs-material && mkdocs build
      • Start Command: cd leaf_flask && gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 120
    5. Add environment variables:
      • SECRET_KEY = any random string
      • OPENAI_API_KEY = your OpenAI key (only if using AI features)
    6. Click Deploy

    Option 2: AWS (EC2 + Elastic Beanstalk or ECS)¶

    Best for organizations already on AWS with IT teams.

    Service Monthly Cost (estimate) Notes
    EC2 t3.micro ~$8/month 1 vCPU, 1 GB RAM, free tier eligible (1 year)
    EC2 t3.small ~$15/month 2 vCPU, 2 GB RAM — recommended
    Elastic Beanstalk EC2 cost + $0 Managed deployment wrapper around EC2
    ECS Fargate ~$15–30/month Serverless containers, no EC2 management
    Application Load Balancer ~$16/month Add if you need custom domain + SSL
    Route 53 (DNS) ~$0.50/month Custom domain routing

    Total estimate: $15–50/month for a basic production setup.

    Quick EC2 Setup¶

    # On a fresh Ubuntu 22.04 EC2 instance
sudo apt update && sudo apt install -y python3-pip python3-venv nginx

# Clone and setup
git clone <repo-url> /opt/iwmi-leaf
cd /opt/iwmi-leaf/leaf_flask
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install mkdocs-material
cd .. && mkdocs build && cd leaf_flask

# Run with Gunicorn
gunicorn app:app --bind 0.0.0.0:5000 --workers 2 --timeout 120 --daemon

# (Optional) Set up Nginx reverse proxy for port 80/443

    Use a process manager

    In production, use systemd or supervisord to keep Gunicorn running and auto-restart on crashes. See the systemd section below.

    Option 3: Google Cloud Platform (GCP)¶

    Service Monthly Cost (estimate) Notes
    Cloud Run ~$5–15/month Serverless containers, pay per request, auto-scaling
    Compute Engine e2-small ~$13/month 2 vCPU, 2 GB RAM
    App Engine (Standard) ~$0–25/month Managed platform, auto-scales to zero

    Cloud Run is the most cost-effective for low-to-moderate traffic.

    Cloud Run Setup¶

    # Build container
docker build -t iwmi-leaf .
docker tag iwmi-leaf gcr.io/YOUR_PROJECT/iwmi-leaf
docker push gcr.io/YOUR_PROJECT/iwmi-leaf

# Deploy
gcloud run deploy iwmi-leaf \
  --image gcr.io/YOUR_PROJECT/iwmi-leaf \
  --port 5000 \
  --memory 1Gi \
  --cpu 1 \
  --allow-unauthenticated

    Best value for money if you're comfortable with basic Linux administration.

    Provider Plan Monthly Cost Specs

    DigitalOcean Regular Droplet $12/month 2 vCPU, 2 GB RAM, 50 GB SSD
    Hetzner CX22 €4/month (~$4.50) 2 vCPU, 4 GB RAM, 40 GB — best value
    DigitalOcean App Platform Basic $5/month Managed platform (like Render)

    Option 6: On-Premises / Government Data Center¶

    For government agencies or organizations with internal hosting infrastructure.

    Requirement Details
    Server Any Linux server with Python 3.10+, 2 GB RAM
    Network HTTP/HTTPS access on ports 80/443
    SSL Use Let's Encrypt (free) or organization's certificate
    Backup Only the data/ directory and ai-docs/ need backup
    Updates Pull from Git and restart the service

    Cost Summary¶

    Scenario Monthly Cost Best For
    Free / testing $0 Demos, evaluation (Render free tier)
    Small production $5–15/month Low traffic, internal use
    Standard production $15–30/month Public-facing, reliable uptime
    With AI features $30–50/month Includes OpenAI API usage (~$5–20/month extra)
    On-premises $0 (infra cost) Government/institutional hosting

    OpenAI API Costs

    The AI recommendation feature calls the OpenAI API per request. Typical cost is $0.01–0.05 per recommendation. With 100 recommendations/month, expect ~$1–5/month in API fees on top of hosting costs. You can also use a self-hosted LLM to avoid this cost entirely.

    • [ ] leaf_flask/data/grampanchayat.* (GP shapefile)

    3. Environment Variables¶

    • [ ] SECRET_KEY — any random string for Flask sessions

    4. Documentation¶

    Production Setup Details¶

    Systemd Service (Linux)¶

    Create /etc/systemd/system/iwmi-leaf.service:

    [Unit]
Description=LEAF DSS Flask Application
After=network.target

[Service]
User=www-data
WorkingDirectory=/opt/iwmi-leaf/leaf_flask
Environment="SECRET_KEY=your-secret-key-here"
Environment="OPENAI_API_KEY=sk-..."
ExecStart=/opt/iwmi-leaf/leaf_flask/venv/bin/gunicorn app:app \
    --bind 127.0.0.1:5000 \
    --workers 2 \
    --timeout 120 \
    --access-logfile /var/log/iwmi-leaf/access.log \
    --error-logfile /var/log/iwmi-leaf/error.log
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

    sudo mkdir -p /var/log/iwmi-leaf
sudo systemctl enable iwmi-leaf
sudo systemctl start iwmi-leaf
sudo systemctl status iwmi-leaf

    Nginx Reverse Proxy¶

    server {
    listen 80;
    server_name iwmi-leaf.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }

    # Optional: serve static files directly for better performance
    location /static/ {
        alias /opt/iwmi-leaf/leaf_flask/static/;
        expires 7d;
    }
}

    Then enable HTTPS with Let's Encrypt:

    sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d iwmi-leaf.yourdomain.com

    Docker (Optional)¶

    Create a Dockerfile in the project root:

    FROM python:3.11-slim

WORKDIR /app

# Install system dependencies for GeoPandas
RUN apt-get update && apt-get install -y \
    libgdal-dev \
    && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
COPY leaf_flask/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt gunicorn

# Copy application
COPY leaf_flask/ ./leaf_flask/
COPY site/ ./site/
COPY ai-docs/ ./ai-docs/

WORKDIR /app/leaf_flask

EXPOSE 5000

CMD ["gunicorn", "app:app", "--bind", "0.0.0.0:5000", "--workers", "2", "--timeout", "120"]

    docker build -t iwmi-leaf .
docker run -p 5000:5000 -e SECRET_KEY=mysecret iwmi-leaf

    Updating the Application¶

    Routine Updates¶

    cd /opt/iwmi-leaf
git pull origin main
cd leaf_flask
source venv/bin/activate
pip install -r requirements.txt   # Only if dependencies changed
cd .. && mkdocs build && cd leaf_flask  # Only if docs changed
sudo systemctl restart iwmi-leaf

    Updating Data¶

    1. Replace the relevant files in leaf_flask/data/
    2. Restart the service — the LRU cache clears on restart
    3. No code changes needed

    Adding a New District's GP Data¶

    1. Prepare a GP-level shapefile (.shp + sidecar files)
    2. Prepare a GP indicator Excel file (same column format as Tinsukia_GP_data.xlsx)
    3. Update data_utils.py to load the new files
    4. Update the gp_districts list in the /api/levels route
    5. Restart the service

    Security Considerations¶

    Area Current State Recommendation for Production
    Authentication None (open API) Add API key or OAuth if needed
    HTTPS Handled by Render/Nginx Always use HTTPS in production
    Secret key Default in code Set via environment variable
    CORS Open to all origins Restrict to your domain if needed
    Rate limiting None Add Flask-Limiter if public-facing
    Input validation Basic Sufficient for current use
    File uploads None No risk
    SQL injection N/A (no database) No risk
