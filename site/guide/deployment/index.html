    Deployment¶

    LEAF DSS can be deployed to cloud platforms (Render recommended) or run locally for development.

    Render (Production)¶

    The app is deployed on Render as a Web Service.

    Configuration¶

    Setting Value
    Runtime Python
    Build Command pip install -r leaf_flask/requirements.txt && pip install mkdocs-material && mkdocs build
    Start Command cd leaf_flask && gunicorn app:app --bind 0.0.0.0:$PORT
    Plan Free or Starter

    Build Command

    The build command installs MkDocs and builds the documentation site into site/, which Flask then serves at /documentation/. If you don't need the docs site, simplify to just pip install -r leaf_flask/requirements.txt.

    Environment Variables¶

    Set these in the Render dashboard under Environment:

    Variable Required Description
    SECRET_KEY Yes Flask secret key (generate a random string)
    OPENAI_API_KEY No Required only for AI recommendations
    PYTHON_VERSION No Python version (e.g. 3.11.0)

    Deploy Steps¶

    1. Push code to GitHub
    2. Create a new Web Service in Render dashboard
    3. Connect your GitHub repository
    4. Set the Build and Start commands as above
    5. Add environment variables
    6. Click Deploy

    Render auto-deploys on every push to the main branch.

    Health Checks¶

    Configure Render's health check to use: 

    GET /health

    Available URLs (Production)¶

    URL Description
    https://iwmi-leaf.onrender.com/ Main dashboard (map UI)
    https://iwmi-leaf.onrender.com/docs Swagger API documentation
    https://iwmi-leaf.onrender.com/documentation/ MkDocs written documentation
    https://iwmi-leaf.onrender.com/api API endpoint listing
    https://iwmi-leaf.onrender.com/health Health check

    Local Development¶

    Quick Start¶

    cd leaf_flask
pip install -r requirements.txt
python app.py

    Runs on http://localhost:5000 with debug mode and auto-reload enabled.

    With Documentation Site¶

    # From project root
pip install mkdocs-material
mkdocs build                    # Generates site/ directory
cd leaf_flask
python app.py                   # Serves docs at /documentation/

    MkDocs Live Preview¶

    For editing documentation with live preview:

    mkdocs serve                    # Serves at http://localhost:8000

    This runs a separate MkDocs dev server with hot-reload — changes to docs/*.md reflect instantly.

    Documentation Site¶

    The MkDocs documentation is served directly from the Flask app at /documentation/. This means both the API and its documentation are available from the same URL.

    How It Works¶

    1. mkdocs build generates a static HTML site in site/
    2. Flask serves these files via the /documentation/ route
    3. MkDocs is configured with site_url: https://iwmi-leaf.onrender.com/documentation/ so all internal links resolve correctly

    Rebuilding Documentation¶

    After editing any file in docs/:

    mkdocs build          # Regenerate site/ directory

    The Flask app serves from site/ — no restart needed (files are served dynamically).

    Configuration¶

    MkDocs configuration is in mkdocs.yml at the project root:

    Setting Value Purpose
    site_url https://iwmi-leaf.onrender.com/documentation/ Ensures internal links work under /documentation/
    theme.name material Material for MkDocs theme
    plugins search Full-text search in documentation

    File Requirements¶

    Ensure these data files are present in leaf_flask/data/:

    Required (core functionality)¶

    File(s) Description
    4DSS_VAR_2.0.shp + .dbf, .shx, .prj Block-level shapefile with indicator data
    Block_assam.shp + .dbf, .shx, .prj District-block mapping shapefile
    DSS_input2.csv Variable metadata and intervention config

    Optional (enhanced features)¶

    File(s) Description Feature
    grampanchayat.shp + .dbf, .shx, .prj GP boundaries (Tinsukia) GP-level views
    Tinsukia_GP_data.xlsx GP indicator data GP variables
    Block_GPs.xlsx GP-to-block mapping GP block grouping
    Vill_Gp_join.csv Village-GP mapping GP code resolution
    districts.geojson District boundaries Map overlay
    protected_areas/Protected_Area_India_Final.shp + .dbf, .shx, .prj, .cst Protected Areas of India Map overlay

    AI features¶

    File(s) Description
    ai-docs/*.pdf Policy documents for RAG pipeline

    Troubleshooting¶

    Issue Solution
    App won't start Check pip install -r leaf_flask/requirements.txt completed without errors
    Shapefile not found Verify .shp, .dbf, .shx, and .prj files all exist in leaf_flask/data/
    AI returns 503 Set OPENAI_API_KEY env var and call POST /api/ai-recommendation/init
    Docs return 404 Run mkdocs build from the project root to generate the site/ directory
    Slow first request Normal — first request loads and caches all shapefiles (~3-5 seconds)
    GP data missing GP data only exists for Tinsukia — ensure GP shapefiles and Excel files are present
    Render deploy fails Check build logs — most common issue is missing Python version or system dependencies
