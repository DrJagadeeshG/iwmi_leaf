LEAF DSS — End-to-End User Workflow
====================================

Mermaid flowchart (paste into any Mermaid renderer — e.g. mermaid.live,
GitHub markdown, or the MkDocs Material site).

```mermaid
flowchart TD
    Start([User opens website]) --> Landing["/  — Main Dashboard<br/>Map of Assam, all districts shown"]

    Landing --> Choose{What does the user want?}
    Choose -->|Explore data| Drill
    Choose -->|Plan clusters| Cluster
    Choose -->|Refresh source data| Update
    Choose -->|Read docs| Docs["/documentation/<br/>MkDocs site"]

    %% ---------- DATA EXPLORATION BRANCH ----------
    subgraph Drill ["Drill-down exploration"]
        D1["Click a district on map<br/>→ /district"] --> D2["See all blocks in district<br/>color-coded by feasibility"]
        D2 --> D3["Click a block<br/>→ /district/block"]
        D3 --> D4["Block detail<br/>variable charts, GPs inside"]
        D4 --> D5["Click a GP<br/>→ /district/block/gp"]
        D5 --> D6["GP-level data<br/>villages, indicators"]
    end

    Drill --> Filters["Apply filters & weights<br/>(side panel)"]
    Filters --> Calc["POST /api/calculate-feasibility<br/>or /api/gp/calculate-feasibility"]
    Calc --> ColorMap["Map recolors:<br/>High / Medium / Low feasibility"]

    ColorMap --> NextAction{Next?}
    NextAction -->|Get AI advice| AI["POST /api/ai-recommendation<br/>RAG over docs + current filters"]
    AI --> AIOut["Recommendation text<br/>+ suggested interventions"]
    NextAction -->|Export| Export["POST /api/export/csv<br/>download filtered dataset"]
    NextAction -->|Move to clustering| Cluster
    NextAction -->|Keep exploring| Drill

    %% ---------- CLUSTERING BRANCH ----------
    subgraph Cluster ["Cluster planning workspace"]
        C1["/clustering or<br/>/district/block/clustering"] --> C2["Pick block + commodity<br/>(Dairy, Goatery, ...)"]
        C2 --> C3["Adjust params:<br/>min/max members, villages, radius"]
        C3 --> C4["GET /api/clusters<br/>seed-and-grow algorithm runs"]
        C4 --> C5["Map shows clusters<br/>+ village pins + centroids"]
        C5 --> C6{Happy with result?}
        C6 -->|No| C3
        C6 -->|Yes| C7["Pick a cluster"]
        C7 --> C8{Action?}
        C8 -->|Find nearest infra| C9["GET /api/infrastructure/nearest<br/>milk plants, vet clinics, markets"]
        C8 -->|Download report| C10["GET /api/clusters/cluster_id/report"]

    end

    %% ---------- UPDATE BRANCH ----------
    subgraph Update ["Source data update"]
        U1["/update"] --> U2["Edit Google Sheets<br/>(block values, intervention defs)"]
        U2 --> U3["POST /api/config/refresh<br/>re-pull sheet contents"]
        U3 --> U4["GET /api/config/sheets-status<br/>verify ingestion"]
    end
    U4 --> Landing

    Docs --> Landing

    PreCheck["⚠️ Pre-check:<br/>SHG GPS duplicate report<br/>(must run before clustering<br/>is trustworthy)"]:::warn
    PreCheck -.feeds clean lat/long.-> C4

    classDef warn fill:#ffe9c7,stroke:#c47a00,color:#5a3a00
```

How to read it
--------------

Three top-level branches off the landing page: explore data, plan clusters,
refresh source data.

- Drill-down is geographic: District → Block → GP (matches the route
  hierarchy in app.py lines 202–238).
- Filters trigger /api/calculate-feasibility, which recolors the map with
  High / Medium / Low scores.
- Clustering is its own workspace — the user picks block + commodity, tweaks
  the 6 parameters, and gets clusters back. They can iterate, then finalize
  a cluster which hands it over to the production-tool dashboard.
- Update page is for the data team: edit Google Sheets, hit refresh, verify
  ingestion.
- The dotted warning box is the GPS-duplicate pre-check requested in the
  May-2026 voice notes — upstream of clustering, not yet built, feeds clean
  coordinates into /api/clusters.
