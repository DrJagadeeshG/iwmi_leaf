    AI Recommendations API¶

    AI-powered, context-aware recommendations using Retrieval-Augmented Generation (RAG). The system combines block-level data with policy documents to generate actionable, citation-backed recommendations for specific interventions.

    Prerequisites

    The AI recommendation feature requires:

    • OPENAI_API_KEY environment variable set on the server
    • RAG dependencies installed (openai, chromadb, langchain, langchain-openai)
    • Vector store initialized (call /api/ai-recommendation/init at least once)
    • PDF policy documents placed in the ai-docs/ directory

    All other LEAF DSS features work without these prerequisites.

    Architecture¶

    User Request
  ↓
Block Data (name, district, intervention, metrics, score)
  ↓
ChromaDB Vector Store ← PDF Policy Documents (ai-docs/)
  ↓
Relevant Policy Context Retrieved (similarity search)
  ↓
OpenAI GPT generates recommendation with context
  ↓
Response: recommendation text + source document citations

    The RAG pipeline:

    1. Ingestion — PDFs in ai-docs/ are split into chunks and embedded into ChromaDB using OpenAI embeddings
    2. Retrieval — When a recommendation is requested, the system queries ChromaDB for chunks most relevant to the block's context
    3. Generation — Retrieved context + block data are sent to OpenAI GPT, which generates a tailored recommendation
    4. Citation — Source documents and page numbers are returned alongside the recommendation

    Get AI Recommendation¶

    Generates a context-aware recommendation for a specific block and intervention. The recommendation considers the block's actual variable values, which criteria are met vs. not met, and relevant policy document context.

    POST /api/ai-recommendation

    Request Body¶

    {
  "block_name": "Digboi",
  "district_name": "Tinsukia",
  "intervention": "Organic Farming",
  "feasibility_score": 65.5,
  "metrics": [
    {
      "label": "Agricultural Land %",
      "value": 45.2,
      "in_range": true,
      "min": 30,
      "max": 70
    },
    {
      "label": "Water Availability",
      "value": 15.0,
      "in_range": false,
      "min": 20,
      "max": 50
    }
  ],
  "filters": [
    {
      "column": "AD",
      "min_val": 30,
      "max_val": 70,
      "weight": 1.0
    }
  ]
}
    Field Type Required Description
    block_name string No Block name (default "Unknown")
    district_name string No District name (default "Unknown")
    intervention string Yes Selected intervention name
    feasibility_score number No Current feasibility score (0–100)
    metrics array No Variable metrics with in-range status — helps the AI understand which criteria pass/fail
    metrics[].label string — Human-readable variable name
    metrics[].value number — Current value for this block
    metrics[].in_range boolean — Whether value falls in acceptable range
    metrics[].min number — Minimum threshold
    metrics[].max number — Maximum threshold
    filters array No Active filter configurations

    Response¶

    {
  "recommendation": "Based on the analysis of Digboi block in Tinsukia district, Organic Farming shows moderate feasibility (65.5%). The block has good agricultural diversity (45.2%) within the acceptable range. However, water availability (15.0%) falls below the minimum threshold of 20%. Recommendations: 1) Invest in water harvesting structures to improve availability. 2) Focus on drought-resistant organic crop varieties...",
  "sources": [
    {
      "document": "Assam_Agriculture_Policy_2023.pdf",
      "page": 12,
      "relevance": 0.89
    },
    {
      "document": "NABARD_Organic_Farming_Guidelines.pdf",
      "page": 5,
      "relevance": 0.82
    }
  ]
}
    Field Type Description
    recommendation string Generated recommendation text (Markdown-formatted)
    sources array Policy documents cited in the recommendation
    sources[].document string PDF filename
    sources[].page integer Page number in the PDF
    sources[].relevance number Similarity score (0–1)

    Errors¶

    Code Description
    503 RAG service unavailable — dependencies not installed or API key missing
    500 Processing error during recommendation generation

    Example¶

    curl -X POST https://iwmi-leaf.onrender.com/api/ai-recommendation \
  -H "Content-Type: application/json" \
  -d '{
    "block_name": "Digboi",
    "district_name": "Tinsukia",
    "intervention": "Organic Farming",
    "feasibility_score": 65.5,
    "metrics": [
      {"label": "Agricultural Land %", "value": 45.2, "in_range": true, "min": 30, "max": 70},
      {"label": "Water Availability", "value": 15.0, "in_range": false, "min": 20, "max": 50}
    ]
  }'

    import requests

response = requests.post(
    "https://iwmi-leaf.onrender.com/api/ai-recommendation",
    json={
        "block_name": "Digboi",
        "district_name": "Tinsukia",
        "intervention": "Organic Farming",
        "feasibility_score": 65.5,
        "metrics": [
            {"label": "Agricultural Land %", "value": 45.2,
             "in_range": True, "min": 30, "max": 70},
            {"label": "Water Availability", "value": 15.0,
             "in_range": False, "min": 20, "max": 50}
        ]
    }
)

if response.status_code == 200:
    data = response.json()
    print("Recommendation:")
    print(data["recommendation"])
    print("\nSources:")
    for src in data.get("sources", []):
        print(f"  - {src['document']} (p.{src['page']}, relevance {src['relevance']:.2f})")
else:
    print(f"Error {response.status_code}: {response.json()}")

    Initialize Vector Store¶

    Rebuilds the RAG vector store by processing all PDF policy documents from the ai-docs/ directory. This must be called at least once before the first AI recommendation request.

    The process:

    1. Scans ai-docs/ for PDF files
    2. Extracts text from each PDF using pypdf
    3. Splits text into overlapping chunks
    4. Generates embeddings using OpenAI's embedding model
    5. Stores embeddings in ChromaDB (local persistent storage)
    POST /api/ai-recommendation/init

    Request Body¶

    No body required.

    Response¶

    {
  "success": true
}

    Errors¶

    Code Response Description
    503 {"success": false, "error": "..."} RAG dependencies not installed
    500 {"success": false, "error": "..."} Error processing PDF documents

    Example¶

    curl -X POST https://iwmi-leaf.onrender.com/api/ai-recommendation/init

    import requests

r = requests.post("https://iwmi-leaf.onrender.com/api/ai-recommendation/init")
result = r.json()
if result['success']:
    print("Vector store initialized successfully")
else:
    print(f"Error: {result.get('error')}")

    When to Re-initialize

    Re-run initialization whenever you:

    • Add new PDF documents to ai-docs/
    • Update or replace existing PDFs
    • Remove documents from the directory
    • Upgrade the embedding model

    The process takes 30–120 seconds depending on the number and size of PDFs.
