    Config & Health API¶

    Application configuration, data level information, map overlay data, and health monitoring. These endpoints provide metadata about the system itself rather than the underlying geospatial data.

    Get Application Configuration¶

    Returns color schemes, feasibility color thresholds, and map configuration used by the frontend. Useful for building custom clients that match the official LEAF DSS color palette.

    GET /api/config

    Response¶

    {
  "colors": {
    "primary": "#2E7D32",
    "secondary": "#1565C0",
    "accent": "#FF9800"
  },
  "feasibility_colors": {
    "very_high": "#1b5e20",
    "high": "#81c784",
    "moderate_high": "#c5e1a5",
    "moderate": "#ffd700",
    "low": "#ff8c00",
    "very_low": "#ff0000",
    "no_data": "#E0E0E0"
  },
  "map_config": {
    "center": [26.5, 93.0],
    "zoom": 7,
    "max_zoom": 18
  }
}
    Field Type Description
    colors object Brand color palette (primary green, secondary blue, accent orange)
    feasibility_colors object Color mapping for each feasibility class
    map_config object Default map center (lat/lng), zoom level, and max zoom

    Example¶

    curl https://iwmi-leaf.onrender.com/api/config

    import requests

r = requests.get("https://iwmi-leaf.onrender.com/api/config")
config = r.json()
print(f"Map center: {config['map_config']['center']}")
print(f"Feasibility colors: {len(config['feasibility_colors'])} classes")

    const r = await fetch('/api/config');
const config = await r.json();

// Initialize map with server-provided config
const map = L.map('map').setView(config.map_config.center, config.map_config.zoom);

    Get Available Data Levels¶

    Returns the data hierarchy levels (block, GP) and which districts support each level. Use this to determine whether to show GP-level options in your UI.

    GET /api/levels

    Response¶

    {
  "levels": [
    {
      "id": "block",
      "name": "Block",
      "available": true
    },
    {
      "id": "gp",
      "name": "Gram Panchayat",
      "available": true,
      "districts": ["Tinsukia"]
    }
  ],
  "gp_districts": ["Tinsukia"]
}
    Field Type Description
    levels array Available data levels
    levels[].id string Level identifier (block or gp)
    levels[].name string Display name
    levels[].available boolean Whether this level has data loaded
    levels[].districts array Districts supporting this level (GP only)
    gp_districts array Shortcut — list of districts with GP-level data

    Example¶

    curl https://iwmi-leaf.onrender.com/api/levels

    import requests

r = requests.get("https://iwmi-leaf.onrender.com/api/levels")
data = r.json()

for level in data['levels']:
    districts = level.get('districts', ['All'])
    print(f"{level['name']}: available={level['available']}, districts={districts}")

    Get Protected Areas GeoJSON¶

    Returns Protected Areas of India (national parks, wildlife sanctuaries, conservation reserves, community reserves, and tiger reserves) as a GeoJSON FeatureCollection. This dataset covers all of India (~550 protected areas) and is used as a map overlay layer to show conservation zones alongside agricultural feasibility data.

    GET /api/protected-areas/geojson

    Response¶

    {
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "properties": {
        "name": "Kaziranga National Park",
        "Type": "National Park",
        "State": "Assam",
        "Area": 858.98,
        "Year": 1974
      },
      "geometry": {
        "type": "Polygon",
        "coordinates": [[[93.1, 26.5], [93.2, 26.5], ...]]
      }
    },
    {
      "type": "Feature",
      "properties": {
        "name": "Manas National Park",
        "Type": "National Park",
        "State": "Assam",
        "Area": 500.0,
        "Year": 1990
      },
      "geometry": { "..." : "..." }
    }
  ]
}
    Field Type Description
    name string Name of the protected area
    Type string Category: National Park, Wildlife Sanctuary, Conservation Reserve, Community Reserve, Tiger Reserve
    State string Indian state where the area is located
    Area number Area in km²
    Year number Year of official designation

    Protected Area Types¶

    Type Count (approx.) Description
    National Park ~100 Highest protection level, no human habitation
    Wildlife Sanctuary ~400 Controlled human activities permitted
    Conservation Reserve ~30 Community-managed conservation areas
    Tiger Reserve ~15 Special tiger conservation zones
    Community Reserve ~5 Community-owned conservation areas

    Use Cases¶

    • Overlay on feasibility map — Identify blocks/GPs that overlap with protected areas, which may affect intervention planning
    • Environmental screening — Filter out areas where certain agricultural activities are restricted
    • Conservation context — Understand the ecological context around proposed intervention areas

    Errors¶

    Code Description
    404 Protected areas shapefile not available
    500 Server error during shapefile processing

    Example¶

    curl https://iwmi-leaf.onrender.com/api/protected-areas/geojson

    import requests
import geopandas as gpd

r = requests.get("https://iwmi-leaf.onrender.com/api/protected-areas/geojson")
geojson = r.json()
print(f"Total protected areas: {len(geojson['features'])}")

gdf = gpd.GeoDataFrame.from_features(geojson['features'])

# Protected areas in Assam
assam = gdf[gdf['State'] == 'Assam']
print(f"\nAssam protected areas: {len(assam)}")
for _, row in assam.iterrows():
    print(f"  {row['name']} ({row['Type']}): {row['Area']:.0f} km²")

# Count by type
print(f"\nBy type:")
print(gdf['Type'].value_counts().to_string())

    const r = await fetch('/api/protected-areas/geojson');
const geojson = await r.json();

// Add as green overlay to Leaflet map
L.geoJSON(geojson, {
  style: {
    fillColor: '#2e7d32',
    fillOpacity: 0.25,
    color: '#1b5e20',
    weight: 1.5
  },
  onEachFeature: (feature, layer) => {
    const p = feature.properties;
    layer.bindTooltip(`<b>${p.name}</b><br>${p.Type}<br>${p.State}`);
  }
}).addTo(map);

    Geometry Simplification

    Geometries are simplified (tolerance 0.005) for web performance. For high-resolution boundaries, access the raw shapefile at leaf_flask/data/protected_areas/Protected_Area_India_Final.shp.

    Health Check¶

    Returns service health status. Use this for uptime monitoring and load balancer health checks.

    GET /health

    Response¶

    {
  "status": "healthy"
}

    Example¶

    curl https://iwmi-leaf.onrender.com/health

    import requests

r = requests.get("https://iwmi-leaf.onrender.com/health")
print(f"Status: {r.json()['status']}")
print(f"HTTP: {r.status_code}")

    Monitoring

    Point your uptime monitor (UptimeRobot, Render health checks, Pingdom, etc.) at /health for reliable service monitoring. The endpoint has no dependencies — it returns immediately without loading data.
