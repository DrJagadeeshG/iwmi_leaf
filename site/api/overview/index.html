    API Overview¶

    The LEAF DSS API provides RESTful endpoints for geospatial data, feasibility analysis, and AI recommendations. All API routes are prefixed with /api/ except the health check at /health.

    Base URL¶

    Production: https://iwmi-leaf.onrender.com
Local:      http://localhost:5000

    Authentication¶

    The API is open and unauthenticated — no API keys or tokens are required. All endpoints are publicly accessible. The only exception is the AI recommendation endpoint, which requires an OPENAI_API_KEY to be set on the server side (not passed by the client).

    Response Format¶

    All endpoints return JSON (Content-Type: application/json) except the CSV export endpoint which returns text/csv.

    Successful responses return data directly. Errors return:

    {
  "error": "Description of what went wrong"
}

    CORS¶

    Cross-Origin Resource Sharing is enabled for all origins via flask-cors. You can call the API from any frontend application, Jupyter notebook, or script without CORS issues.

    Endpoint Categories¶

    Category Prefix Endpoints Description
    Blocks /api/blocks 4 Block-level geospatial data and lookups
    Locations /api/locations, /api/districts 4 Hierarchical location data (districts, blocks, GPs)
    Gram Panchayats /api/gp 8 GP-level geospatial data and lookups
    Interventions /api/interventions 2 Intervention definitions and variable configs
    Variables /api/variables 3 Variable metadata and statistics
    Feasibility /api/calculate-feasibility 2 Weighted feasibility score calculation
    AI /api/ai-recommendation 2 RAG-powered AI recommendations
    Export /api/export 1 CSV data export
    Config /api/config, /health 5 App config, health, map overlays

    Total: 31 endpoints across 9 categories.

    Common Patterns¶

    GeoJSON Responses¶

    Geospatial endpoints return standard GeoJSON FeatureCollections. Each feature includes a properties object with all variable data and a geometry object with polygon coordinates in WGS84 (EPSG:4326).

    {
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "properties": {
        "Block_name": "Digboi",
        "Dist_Name": "Tinsukia",
        "BLOCK_ID": "1234",
        "AD": 45.2,
        "WA": 30.1
      },
      "geometry": {
        "type": "Polygon",
        "coordinates": [[[95.1, 27.3], [95.2, 27.3], [95.2, 27.4], [95.1, 27.3]]]
      }
    }
  ]
}

    Working with GeoJSON

    GeoJSON responses can be directly loaded into mapping libraries like Leaflet, Mapbox, or QGIS. In Python, use geopandas.GeoDataFrame.from_features(response['features']) to convert to a GeoDataFrame.

    Filter Criteria¶

    POST endpoints that calculate feasibility accept an array of filter criteria:

    {
  "column": "AD",
  "min_val": 20,
  "max_val": 60,
  "weight": 1.0
}
    Field Type Required Description
    column string Yes Variable/column name from the shapefile
    min_val number Yes Minimum acceptable threshold
    max_val number Yes Maximum acceptable threshold
    weight number No Importance weight (0–1), default 1.0

    Logic Modes¶

    When multiple filters are provided, you can choose how they combine:

    Mode Behavior Use Case
    AND (default) Block must meet all criteria to score highly Strict screening — only blocks meeting every requirement
    OR Block scores based on any criteria met Lenient screening — blocks meeting at least one requirement

    Error Handling¶

    HTTP Code Meaning When
    200 Success Normal response
    404 Resource not found Invalid block ID, GP name, or missing data files
    500 Server error Unexpected processing error
    503 Service unavailable AI/RAG dependencies not configured

    Rate Limiting¶

    There are no rate limits on the API. However, some endpoints (particularly GET /api/blocks and POST /api/calculate-feasibility) return large GeoJSON payloads (1–5 MB). Consider caching responses client-side when building frontend applications.

    Quick Examples¶

    Get all blocks and count them (Python)¶

    import requests

r = requests.get("https://iwmi-leaf.onrender.com/api/blocks")
geojson = r.json()
print(f"Total blocks: {len(geojson['features'])}")

# Get unique districts
districts = set(f['properties']['Dist_Name'] for f in geojson['features'])
print(f"Districts: {len(districts)}")

    Run feasibility analysis (JavaScript)¶

    const response = await fetch('/api/calculate-feasibility', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    intervention: 'Organic Farming',
    logic: 'AND'
  })
});
const result = await response.json();
console.log(`Mean score: ${result.statistics.mean_score}`);
console.log(`High feasibility blocks: ${result.statistics.high_count}`);

    Load into GeoPandas (Python)¶

    import requests
import geopandas as gpd

r = requests.get("https://iwmi-leaf.onrender.com/api/blocks")
gdf = gpd.GeoDataFrame.from_features(r.json()['features'])
print(gdf[['Block_name', 'Dist_Name', 'AD', 'WA']].head())
