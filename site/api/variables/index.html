    Variables API¶

    Metadata and statistics for the data variables (indicators) available in the shapefiles. Variables represent measurable characteristics of each block or GP — agriculture percentages, water indices, infrastructure counts, livestock densities, and more.

    Each variable has a short code (e.g., AD, WA, CF) used as column names in the shapefile and GeoJSON properties. These endpoints provide human-readable labels, descriptions, group categories, and statistical ranges.

    Get All Block-Level Variables¶

    Returns metadata for every numeric variable in the block-level shapefile, including min/max/mean statistics computed across all ~152 blocks.

    GET /api/variables

    Response¶

    [
  {
    "field": "AD",
    "label": "Agricultural Diversity",
    "description": "Percentage of agricultural land diversity",
    "group": "Land & Agriculture",
    "data_min": 0.0,
    "data_max": 95.5,
    "data_mean": 42.3
  },
  {
    "field": "WA",
    "label": "Water Availability",
    "description": "Water availability index",
    "group": "Water Resources",
    "data_min": 5.0,
    "data_max": 88.0,
    "data_mean": 35.7
  }
]
    Field Type Description
    field string Column code in the shapefile (used in filter criteria)
    label string Human-readable name
    description string What this variable measures
    group string Category group for UI organization
    data_min number Minimum value across all blocks
    data_max number Maximum value across all blocks
    data_mean number Mean value across all blocks

    Variable Groups¶

    Variables are organized into thematic groups:

    Group Description Example Variables
    Land & Agriculture Crop patterns, soil quality, agricultural land use AD, CF, CI
    Water Resources Water availability, irrigation, groundwater WA, GW
    Infrastructure Roads, markets, connectivity, facilities IN, RD
    Livestock Animal populations, livestock density LV, CT
    People & Collectives Demographics, SHGs, cooperatives PO, SH

    Example¶

    curl https://iwmi-leaf.onrender.com/api/variables

    import requests
import pandas as pd

r = requests.get("https://iwmi-leaf.onrender.com/api/variables")
variables = r.json()

# Create a summary DataFrame
df = pd.DataFrame(variables)
print(f"Total variables: {len(df)}")
print(f"\nBy group:")
print(df.groupby('group').size().sort_values(ascending=False))
print(f"\nSample:")
print(df[['field', 'label', 'group', 'data_min', 'data_max']].head(10).to_string(index=False))

    const r = await fetch('/api/variables');
const variables = await r.json();

// Group by category
const byGroup = {};
variables.forEach(v => {
  (byGroup[v.group] = byGroup[v.group] || []).push(v);
});

Object.entries(byGroup).forEach(([group, vars]) => {
  console.log(`${group}: ${vars.length} variables`);
});

    GP Variables

    For GP-level variables, use GET /api/gp/variables instead. GP variables share many of the same codes but may have different ranges due to the finer spatial resolution.

    Get Variable Groups¶

    Returns the list of variable group categories used to organize variables in the UI.

    GET /api/variable-groups

    Response¶

    {
  "groups": [
    "Land & Agriculture",
    "Water Resources",
    "Infrastructure",
    "Livestock",
    "People & Collectives"
  ]
}

    Example¶

    curl https://iwmi-leaf.onrender.com/api/variable-groups

    import requests

r = requests.get("https://iwmi-leaf.onrender.com/api/variable-groups")
groups = r.json()['groups']
print(f"Available groups: {', '.join(groups)}")

    Get Variable Statistics¶

    Returns min, max, and mean values for a specific variable column. Supports both block-level and GP-level data via the optional level query parameter.

    GET /api/variable-stats/{variable}

    Parameters¶

    Parameter In Type Required Description
    variable path string Yes Variable column code (e.g. AD, WA, CF)

    Response¶

    {
  "min": 0.0,
  "max": 95.5,
  "mean": 42.3
}
    Field Type Description
    min number Minimum value across all features
    max number Maximum value across all features
    mean number Mean value across all features

    Missing Variables

    If the variable name doesn't exist in the shapefile, the endpoint returns default values {min: 0, max: 100, mean: 50} instead of an error.

    Example¶

    curl https://iwmi-leaf.onrender.com/api/variable-stats/AD

    import requests

# Check stats for multiple variables
variables = ['AD', 'WA', 'CF', 'CI']
for var in variables:
    r = requests.get(f"https://iwmi-leaf.onrender.com/api/variable-stats/{var}")
    stats = r.json()
    print(f"{var}: {stats['min']:.1f} – {stats['max']:.1f} (mean {stats['mean']:.1f})")

    const variable = 'AD';
const r = await fetch(`/api/variable-stats/${variable}`);
const stats = await r.json();
console.log(`${variable}: ${stats.min} – ${stats.max} (avg ${stats.mean.toFixed(1)})`);
