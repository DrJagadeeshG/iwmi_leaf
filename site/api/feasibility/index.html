    Feasibility API¶

    Calculate weighted feasibility scores for agricultural interventions at both block and GP levels. This is the core analytical engine of LEAF DSS — it evaluates how suitable each spatial unit is for a given intervention based on multiple variable criteria.

    How Feasibility Scoring Works¶

    The scoring algorithm follows these steps:

    1. Define criteria — Provide variable filters with acceptable min/max ranges and importance weights
    2. Evaluate each feature — For each block/GP, check if each variable falls within the specified range
    3. Calculate score — Weighted sum of criteria met, normalized to 0–100
    Score = Σ (variable_in_range × weight) / Σ (weight) × 100
    1. Classify — Scores are bucketed into feasibility classes:
    Score Range Class Color
    80–100 Very High Dark Green
    60–80 High Green
    45–60 Moderate High Light Green
    30–45 Moderate Yellow
    15–30 Low Orange
    0–15 Very Low Red
    1. Return — GeoJSON with scores embedded as properties, plus distribution statistics

    Calculate Block Feasibility¶

    Calculates feasibility scores for all blocks (~152) based on provided filters.

    POST /api/calculate-feasibility

    Request Body¶

    {
  "intervention": "Organic Farming",
  "filters": [
    {
      "column": "AD",
      "min_val": 20,
      "max_val": 60,
      "weight": 1.0
    },
    {
      "column": "WA",
      "min_val": 30,
      "max_val": 70,
      "weight": 0.8
    }
  ],
  "logic": "AND",
  "district": "134"
}
    Field Type Required Description
    intervention string No Intervention name — uses its default config if no filters provided
    filters array No Custom filter criteria (overrides intervention defaults if both provided)
    filters[].column string Yes Variable column name (e.g. AD, WA)
    filters[].min_val number Yes Minimum acceptable value
    filters[].max_val number Yes Maximum acceptable value
    filters[].weight number No Importance weight (0–1), default 1.0
    logic string No "AND" or "OR", default "AND"
    district string No District ID to filter statistics to a specific district

    Intervention vs Filters

    • Intervention only: System uses the intervention's pre-configured variables from the CSV
    • Filters only: System uses your custom filters
    • Both: Custom filters take precedence, intervention is ignored
    • Neither: All blocks scored as "no data"

    Response¶

    {
  "geojson": {
    "type": "FeatureCollection",
    "features": [
      {
        "type": "Feature",
        "properties": {
          "Block_name": "Digboi",
          "Dist_Name": "Tinsukia",
          "AD": 45.2,
          "WA": 35.1,
          "feasibility_score": 75.5,
          "feasibility_class": "High",
          "feasibility_color": "#81c784"
        },
        "geometry": { "type": "Polygon", "coordinates": [[...]] }
      }
    ]
  },
  "statistics": {
    "mean_score": 52.3,
    "very_high_count": 12,
    "high_count": 33,
    "moderate_high_count": 28,
    "moderate_count": 35,
    "low_count": 25,
    "very_low_count": 19
  },
  "filters_applied": [
    { "column": "AD", "min_val": 20, "max_val": 60, "weight": 1.0 },
    { "column": "WA", "min_val": 30, "max_val": 70, "weight": 0.8 }
  ]
}
    Field Type Description
    geojson object GeoJSON FeatureCollection with all blocks and feasibility properties
    geojson.features[].properties.feasibility_score number Score 0–100
    geojson.features[].properties.feasibility_class string Class label
    geojson.features[].properties.feasibility_color string Hex color for map display
    statistics object Distribution counts and mean score
    filters_applied array Echo of the filters used in calculation

    Example¶

    # Using intervention defaults
curl -X POST https://iwmi-leaf.onrender.com/api/calculate-feasibility \
  -H "Content-Type: application/json" \
  -d '{"intervention": "Organic Farming", "logic": "AND"}'

# Using custom filters
curl -X POST https://iwmi-leaf.onrender.com/api/calculate-feasibility \
  -H "Content-Type: application/json" \
  -d '{
    "filters": [
      {"column": "AD", "min_val": 20, "max_val": 60, "weight": 1.0},
      {"column": "WA", "min_val": 30, "max_val": 70, "weight": 0.8}
    ],
    "logic": "AND"
  }'

    import requests

# Calculate feasibility with custom filters
response = requests.post(
    "https://iwmi-leaf.onrender.com/api/calculate-feasibility",
    json={
        "filters": [
            {"column": "AD", "min_val": 20, "max_val": 60, "weight": 1.0},
            {"column": "WA", "min_val": 30, "max_val": 70, "weight": 0.8}
        ],
        "logic": "AND"
    }
)
result = response.json()

print(f"Mean feasibility: {result['statistics']['mean_score']:.1f}")
print(f"High feasibility blocks: {result['statistics'].get('high_count', 0)}")

# Find top-scoring blocks
features = result['geojson']['features']
top = sorted(features, key=lambda f: f['properties'].get('feasibility_score', 0), reverse=True)
for f in top[:5]:
    p = f['properties']
    print(f"  {p['Block_name']}: {p['feasibility_score']:.1f} ({p['feasibility_class']})")

    const response = await fetch('/api/calculate-feasibility', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    intervention: 'Organic Farming',
    logic: 'AND'
  })
});
const result = await response.json();

console.log(`Mean score: ${result.statistics.mean_score}`);
// Add scored GeoJSON to map
L.geoJSON(result.geojson, {
  style: feature => ({
    fillColor: feature.properties.feasibility_color,
    fillOpacity: 0.7
  })
}).addTo(map);

    Calculate GP Feasibility¶

    Calculates feasibility scores at the Gram Panchayat level. Uses the same scoring algorithm as block feasibility but operates on GP-level data.

    POST /api/gp/calculate-feasibility

    Request Body¶

    {
  "filters": [
    {
      "column": "AD",
      "min_val": 20,
      "max_val": 60,
      "weight": 1.0
    }
  ],
  "logic": "AND",
  "block": "Digboi"
}
    Field Type Required Description
    filters array Yes Filter criteria (same format as block feasibility)
    logic string No "AND" or "OR", default "AND"
    block string No Block name to filter — only calculates for GPs in this block

    Block Filter

    The block parameter is useful for focused analysis. Without it, all ~95 GPs in Tinsukia are scored. With it, only GPs in the specified block are included.

    Response¶

    Same format as block feasibility — GeoJSON with scores and distribution statistics.

    Errors¶

    Code Description
    404 GP data not available

    Example¶

    curl -X POST https://iwmi-leaf.onrender.com/api/gp/calculate-feasibility \
  -H "Content-Type: application/json" \
  -d '{
    "filters": [{"column": "AD", "min_val": 20, "max_val": 60, "weight": 1.0}],
    "logic": "AND",
    "block": "Digboi"
  }'

    import requests

r = requests.post(
    "https://iwmi-leaf.onrender.com/api/gp/calculate-feasibility",
    json={
        "filters": [
            {"column": "AD", "min_val": 20, "max_val": 60, "weight": 1.0}
        ],
        "block": "Digboi"
    }
)
result = r.json()
print(f"GPs scored: {len(result['geojson']['features'])}")
print(f"Mean score: {result['statistics']['mean_score']:.1f}")
