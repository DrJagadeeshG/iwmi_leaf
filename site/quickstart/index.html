    Getting Started¶

    This guide walks you through installing LEAF DSS locally, understanding the project layout, and making your first API calls.

    Prerequisites¶

    • Python 3.10+ (3.11 recommended)
    • pip (included with Python)
    • Git for cloning the repository

    Installation¶

    # Clone the repository
git clone <repo-url>
cd IWMI_LEAF

# Install Flask app dependencies
cd leaf_flask
pip install -r requirements.txt

# Run the development server
python app.py

    The app starts at http://localhost:5000 with debug mode enabled. Hot-reload is active — code changes auto-restart the server.

    MkDocs Documentation

    To also serve the documentation site at /documentation/: 

    # From the project root (not leaf_flask/)
pip install mkdocs-material
mkdocs build          # generates site/ directory
python leaf_flask/app.py   # docs now served at /documentation/

    Key URLs¶

    URL Description
    http://localhost:5000/ Main dashboard — interactive map with feasibility analysis
    http://localhost:5000/docs Swagger UI — interactive API explorer
    http://localhost:5000/documentation/ MkDocs documentation site (this site)
    http://localhost:5000/api API endpoint listing (JSON)
    http://localhost:5000/health Health check endpoint

    Project Structure¶

    IWMI_LEAF/
├── leaf_flask/                 # Flask application
│   ├── app.py                  # All API routes + page routes
│   ├── config.py               # Colors, variable groups, map config
│   ├── data_utils.py           # Shapefile/CSV loading, GeoJSON conversion
│   ├── feasibility.py          # Feasibility score calculations
│   ├── rag_utils.py            # AI/RAG recommendation engine
│   ├── requirements.txt        # Python dependencies
│   ├── templates/
│   │   └── index.html          # Main dashboard template (Jinja2)
│   ├── static/
│   │   ├── css/style.css       # All application styles
│   │   ├── js/app.js           # Frontend logic (map, filters, charts)
│   │   └── images/             # Logos (IWMI, CGIAR, GIZ)
│   └── data/                   # Data files (not in git)
│       ├── 4DSS_VAR_2.0.shp    # Block-level shapefile
│       ├── Block_assam.shp     # District-block mapping
│       ├── grampanchayat.shp   # GP-level shapefile (Tinsukia)
│       ├── districts.geojson   # District boundaries
│       ├── protected_areas/    # Protected Areas of India shapefile
│       ├── DSS_input2.csv      # Variable metadata & intervention config
│       ├── Block_GPs.xlsx      # GP-to-block mapping
│       └── Tinsukia_GP_data.xlsx  # GP indicator data
├── ai-docs/                    # PDF policy documents for RAG
├── docs/                       # MkDocs source files
├── site/                       # MkDocs built output (generated)
├── mkdocs.yml                  # MkDocs configuration
└── requirements.txt            # Root-level dependencies (Streamlit, legacy)

    Environment Variables¶

    Variable Required Description Default
    SECRET_KEY No Flask secret key for sessions iwmi-leaf-secret-key
    OPENAI_API_KEY No OpenAI API key (required for AI recommendations) —
    PORT No Server port (used by Render) 5000

    Note

    The app works fully without OPENAI_API_KEY — only the AI recommendation feature requires it. All other endpoints (map, feasibility, export) work out of the box.

    First API Calls¶

    Health check¶

    curl http://localhost:5000/health
# → {"status": "healthy"}

    List available interventions¶

    curl http://localhost:5000/api/interventions

    Get all blocks as GeoJSON¶

    curl http://localhost:5000/api/blocks | python -m json.tool | head -20

    Calculate feasibility for an intervention¶

    curl -X POST http://localhost:5000/api/calculate-feasibility \
  -H "Content-Type: application/json" \
  -d '{"intervention": "Organic Farming", "logic": "AND"}'

    Get protected areas overlay¶

    curl http://localhost:5000/api/protected-areas/geojson | python -m json.tool | head -30

    Using the Dashboard¶

    1. Select an Intervention¶

    Choose an intervention from the dropdown (e.g., "Organic Farming"). The map immediately colors all blocks by feasibility score.

    2. Filter by District¶

    Optionally select a district to zoom in and filter the distribution chart to that district's blocks only.

    3. Configure Variables¶

    Click Configure to open the modal. Adjust variable ranges (drag sliders), weights, and add/remove variables. Click Apply to recalculate.

    4. Toggle Map Overlays¶

    Check the Protected Areas checkbox above the map to overlay national parks, wildlife sanctuaries, and conservation reserves.

    5. Explore Block Details¶

    Click any block on the map to open its detail view. See all variable values grouped by category (Land & Agriculture, Water, Infrastructure, Livestock, People & Collectives), with out-of-range indicators.

    6. Drill Down to GPs¶

    When viewing a block in Tinsukia district, a GP dropdown appears. Select a GP to see village-level indicators.

    7. Get AI Recommendations¶

    In the block detail view, click the robot icon to generate an AI-powered recommendation backed by policy document citations.

    8. Export Data¶

    Click the CSV button in the filter bar to download the current dataset (with feasibility scores if an intervention is selected).

    Using the Swagger UI¶

    1. Navigate to http://localhost:5000/docs
    2. Endpoints are grouped by tag: Blocks, Locations, GPs, Interventions, Variables, Feasibility, AI, Export, Config
    3. Click any endpoint to expand it and see parameters + response schema
    4. Click Try it out to enable editing
    5. Fill in parameters and click Execute to make a live request
    6. View the response body, headers, and cURL command

    POST Endpoints

    For POST endpoints (Feasibility, AI, Export), edit the JSON body in the text area before clicking Execute. The Swagger UI pre-fills a template based on the schema.

    Troubleshooting¶

    Problem Solution
    ModuleNotFoundError Run pip install -r leaf_flask/requirements.txt
    Shapefile not found Ensure data files exist in leaf_flask/data/
    AI recommendations return 503 Set OPENAI_API_KEY and run /api/ai-recommendation/init
    Documentation 404 Run mkdocs build from the project root to generate site/
    Map shows no blocks Check that 4DSS_VAR_2.0.shp and associated .dbf/.shx files exist
    GP dropdown missing GP data is only available for Tinsukia — select that district first
